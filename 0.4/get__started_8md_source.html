17 ## Download ##
18 
19 tinyusb uses github as online repository https://github.com/hathach/tinyusb since it is the best place for open source project.
20 
21 If you are using Linux, you already know how to what to do. But If Windows is your OS, I would suggest to install [git](http://git-scm.com/) and front-end gui such as [tortoisegit](http://code.google.com/p/tortoisegit) to begin with.
22 
23 After downloading/cloning, the code base is composed of
24 
25 Folder | Description
26 ----- | -------------
27 boards | Source files of supported boards
28 demos | Source & project files for demonstration application
29 mcu | Low level mcu core & peripheral drivers (e.g CMSIS )
30 tests | Unit tests for the stack
31 tinyusb | All sources files for tinyusb stack itself.
32 vendor | Source files from 3rd party such as freeRTOS, fatfs etc ...
33 
34 ## Import and Build ##
35 
36 *repo/demos* is the folder where all the application & project files are located. There are demos for both device and hosts. For each, there are different projects for each of supported RTOS.
37 
38 ### Prerequisites ###
39 
40 In order to build and run application demo, you would need
41 
42 - A [supported development board](../../boards/readme.md)
43 - A supported toolchain: LPCXpresso, Keil, IAR.
44 
45 ### LPCXpresso ###
46 
47 To prevent any sort of problems, it is recommended to do **EXACTLY** as follows (esp the item 4)
48 
49 1. Copy the whole repo folder to your lpcxpresso's workspace.
50 2. Click *File->Import*, then expand the *General* folder and select **Existing Projects into Workspace** and click Next.
51 
52  ![lpcxpresso_import.png](http://docs.tinyusb.org/images/lpcxpresso_import.png)
53 
54 3. On the next dialog, Click *Browse* and choose the **repo/demos** folder inside the repo. You should see a list of all demo applications.
55 
56  ![lpcxpresso_import2.png](http://docs.tinyusb.org/images/lpcxpresso_import2.png)
57 
58 4. **IMPORTANT** make sure the option **Copy projects into workspace** is **CLEAR**. As demo application uses link folders and this option may cause problem for the import.
59 5. Choose any of the demo application and click *Finish*.
60 6. Select the configure corresponding to your development board.
61 
62  ![lpcxpresso_select_board.png](http://docs.tinyusb.org/images/lpcxpresso_select_board.png)
63 
64 7. Then select the correct MCU option from project properties then you are ready to go.
65 
66  ![lpcxpresso_mcu.png](http://docs.tinyusb.org/images/lpcxpresso_mcu.png)
67 
68 ### Keil ###
69 
70 It is relatively simple for Keil

72 1. Open the desired demo project e.g *demos/host/host_freertos/host_freertos.uvproj*
73 2. Select the configure corresponding to your development board and build it.
74 
75  ![keil_select_board.png](http://docs.tinyusb.org/images/keil_select_board.png)

77 ### IAR ###
78 
79 IAR is just as easy as Keil
80 
81 1. Open the desired demo project e.g *demos/host/host_freertos/host_freertos.eww*
82 2. Again select the configure corresponding to your development board and build it.

86 ## Configure demo ##
87 
88 Application demo is written to have the code excluded if its required option is not enabled in [tusb_config.h](). Some of combination may exceed the 32KB limit of IAR/Keil so you may want to re-configure to disable some class support, decrease TUSB_CFG_DEBUG or increase the compiler optimization level.
89 
90 In addition, there are some configuration you can change such as
91 
92 - CFG_UART_BAUDRATE in board.h
93 - CFG_PRINTF_TARGET in the specific board header (e.g board_ea4357.h) to either Semihost, Uart, or SWO.
